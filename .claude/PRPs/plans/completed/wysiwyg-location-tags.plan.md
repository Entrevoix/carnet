# Plan: Capture Metadata Suite — WYSIWYG editing, Location picker, Tag system

> ✅ SHIPPED — capture-metadata suite complete: Task 0 frontmatter (#36 c4bee19), F3 Tags (#38 cf45a24 + #39 fixes), F2 Location (#40 787688b). Archived.

## Summary
Three related capture/editing features for the **React Native + Expo** carnet app
(NOT the Kotlin/Compose stack the original spec assumed): (1) WYSIWYG note editing,
(2) a location picker that stores `lat,lon` in frontmatter, (3) a first-class tag
system backed by YAML frontmatter. All three read/write the markdown frontmatter
block, so they share a new **frontmatter/metadata subsystem** built as Task 0.

## User Story
As a carnet user, I want to edit notes visually, attach a location, and tag captures,
so that my notes are richer and browsable — while the on-disk markdown stays clean and
Obsidian-compatible.

## Problem → Solution
Today captures are raw text → enriched by the LLM (OmniRoute) into markdown with
LLM-generated frontmatter; the app never composes frontmatter itself, has no
array/upsert frontmatter helpers, indexes nothing about the vault, and edits notes as
raw markdown in a `TextInput`. → Add a deterministic frontmatter metadata layer, a
WYSIWYG editor (already largely built), a location field, and a tag index + browser.

## ⚠️ Architecture realities discovered (read before planning any task)

1. **Frontmatter is generated by the LLM, not the app.** `src/lib/prompts.ts` system
   prompts instruct OmniRoute to emit the `---…---` block per mode
   (`buildIdeaPrompt`/`buildJournalPrompt`/`buildPersonPrompt`, prompts.ts:44-124). The
   idea/journal/person templates ALREADY emit `tags: [idea, seedling, {tag1}, {tag2}]`
   etc. ⇒ **User-entered location/tags must be merged into the returned markdown
   *deterministically, after enrichment, before write* — in BOTH the online path
   (`confirmSave`, CaptureScreen.tsx:329-394) and the offline drain
   (`processRow`, queue.ts:245-269).** Do not rely on the LLM to honor injected values.

2. **No YAML library** (`package.json` has no `yaml`/`gray-matter`). Frontmatter is
   hand-rolled line-scanning in `writer.ts`. Existing helpers:
   - `extractFrontmatterField(md, field): string | null` (writer.ts:610) — single scalar read.
   - `rewriteFrontmatterField(content, field, value): string` (writer.ts:665) — rewrites an
     **existing** field and **throws `field not present`** if absent (writer.ts:709-711);
     rejects newlines in value; no array support.
   - `splitFrontmatter(md): { header, body }` (writer.ts:651) — byte-exact `header+body===md`.
   - `stripFrontmatter` (630), `upsertSection` (325, body `##` sections — NOT frontmatter).
   ⇒ **Array (`tags`) and add-if-missing (`location`) handling do not exist and must be
   built** (Task 0).

3. **Recents = AsyncStorage history, max 20, NOT a vault scan** (`storage.ts`,
   `HISTORY_KEY="carnet:history:v1"`, `getRecentCaptures`). `CaptureEntry` =
   `{ id, mode, title, filepath, createdAt }` (storage.ts:8-14) — no tags/location.
   ⇒ **Tag autocomplete, the tag browser, and tag filtering need NEW vault enumeration**
   (list note files + read each frontmatter). A private lister exists
   (`listChildNames`, writer.ts:122) but is not exported and only lists one dir.

4. **Capture input is PRE-LLM raw text, not markdown.** The idea/journal/person inputs
   are plain `TextInput`s feeding the enrichment LLM (CaptureScreen ModeInput,
   ~600-752). ⇒ **WYSIWYG at *capture* time is semantically questionable** (you're
   typing raw thoughts, not markdown). WYSIWYG belongs in the *detail edit* of a saved
   note (already built). See Feature 1 scope note.

## Shared subsystem (Task 0) — `src/lib/frontmatter.ts`

A small, pure, fully-unit-tested module generalizing the hand-rolled helpers. No YAML lib.
All functions operate on the markdown string and preserve the rest byte-for-byte where possible.

```ts
// Parse the --- block into ordered [key, rawValue] pairs (rawValue = text after `key:`).
export function parseFrontmatter(md: string): { fields: Array<[string,string]>; hasBlock: boolean };

// Upsert a SCALAR field: rewrite if present, else INSERT before the closing ---.
// (fills the gap rewriteFrontmatterField leaves — it throws when absent.)
export function upsertFrontmatterField(md: string, field: string, value: string): string;

// tags: parse `tags: [a, b]` (inline flow array) AND `tags:\n  - a\n  - b` (block); return string[].
export function getFrontmatterTags(md: string): string[];
// set/merge tags as an inline flow array `tags: [a, b, c]` (Obsidian-compatible), upserting the field.
export function setFrontmatterTags(md: string, tags: string[]): string;

// normalize one tag: trim, lowercase, spaces→-, strip leading '#', collapse repeats, [a-z0-9-] only.
export function normalizeTag(raw: string): string;
```
GOTCHA: notes with **no frontmatter** (`splitFrontmatter` returns `header:""`) — `upsert*`
must CREATE a `---\n…\n---\n` block in that case. GOTCHA: idea/journal already have a
`tags:` line from the LLM — merge (dedupe) rather than overwrite. Preserve existing tags.

## Testing Strategy

### Unit Tests (vitest, node env — pure logic only)
| Test | Input | Expected | Edge |
|---|---|---|---|
| `upsertFrontmatterField` add-if-missing | md w/o `location` | inserts before closing `---` | no frontmatter at all → creates block |
| `upsertFrontmatterField` rewrite | md with `location: x` | replaces value | preserves other fields byte-exact |
| `getFrontmatterTags` | `tags: [a, b]` AND block list | `["a","b"]` | missing tags → `[]` |
| `setFrontmatterTags` merge | LLM `tags:[idea]` + user `[work]` | `[idea, work]` deduped | dup/case → normalized once |
| `normalizeTag` | `"#My Tag"` | `"my-tag"` | emoji/punctuation stripped |
| `formatCoords`/`parseCoords` | `{38.9072,-77.0369}` | `"38.9072,-77.0369"` ↔ back | bad string → null |
| `buildTagIndex` | in-memory notes | counts per tag | note w/o frontmatter ignored |

### Edge Cases Checklist
- [ ] Note with NO frontmatter (upsert must create block)
- [ ] Offline capture carries tags+location through drain
- [ ] LLM-emitted tags merged, not clobbered
- [ ] Large vault scan performance (cache, concurrency cap)
- [ ] Location permission denied / canAskAgain=false
- [ ] Tag normalization idempotent
- [ ] Frontmatter stays byte-identical when only body edited (WYSIWYG)
