# Plan: Rename app identifier `us.beary.carnet` → `com.ventoux.carnet`

## Summary
Re-namespace the application identifier across all three apps (mobile Android, mobile iOS, desktop Tauri) from `us.beary.carnet` to `com.ventoux.carnet`. Because the Expo `android/` and `ios/` projects are gitignored and **regenerated by `expo prebuild` from `app.json` + config plugins** (which all derive the package from `config.android.package`), the functional change is a handful of string edits plus a prebuild + rebuild — not a manual native-file rename.

## User Story
As the app owner, I want the app to ship under the `com.ventoux.*` namespace (matching the org's other app, `com.ventoux.openclaw`), so that the identifier reflects the real org instead of the placeholder `us.beary.carnet`.

## Problem → Solution
Identifier is the placeholder `us.beary.carnet` everywhere → it becomes `com.ventoux.carnet` across mobile (Android + iOS) and the desktop Tauri app, with the native projects regenerated so all emitted Kotlin/manifest/intent references follow automatically.

- **Complexity**: Small edits / Medium operational care (prebuild --clean, full rebuild, new app identity)

### Decisions (confirmed with user)
- **Scope**: Everything — mobile Android **and** iOS bundle id **and** desktop Tauri identifier.
- **Existing data**: **Fresh start.** The rename produces a NEW app identity; Android sandboxes data per package, so the current `us.beary.carnet` app's notes/audio/settings do NOT migrate. User accepts losing them (old app stays installed until manually removed). **No migration step.**

```js
//         withCaptureWidget.js:222, withAppShortcuts.js:123
const packageName = config.android?.package;
// …then emitted Kotlin / manifest / intents all interpolate ${packageName}:
//   `package ${packageName}.audiodecoder`     (withAudioDecoder.js:34)
```

→ Implication: editing `app.json`'s `android.package` is sufficient for all plugin output. Do not touch the plugin bodies (except the stale comment).

→ PR #23 review already flagged this: "PKG_PATH hardcoded — false-red if android.package changes. Derive from app.json." Fix permanently by deriving.

## Step-by-Step Tasks

### Task 1: Change the mobile identifier (source of truth)
- **ACTION**: Edit `apps/mobile/app.json`.
- **IMPLEMENT**: Line 21 `"package": "us.beary.carnet"` → `"package": "com.ventoux.carnet"`; line 12 `"bundleIdentifier": "us.beary.carnet"` → `"bundleIdentifier": "com.ventoux.carnet"`.
- **MIRROR**: N/A (data edit).
- **GOTCHA**: Do NOT change `name`, `slug`, or `scheme`. Only the two identifier strings.
- **VALIDATE**: `node -p "require('./apps/mobile/app.json').expo.android.package"` prints `com.ventoux.carnet`; same for `.expo.ios.bundleIdentifier`.

### Task 2: Change the desktop identifier
- **ACTION**: Edit `apps/desktop/src-tauri/tauri.conf.json`.
- **IMPLEMENT**: Line 5 `"identifier": "us.beary.carnet"` → `"identifier": "com.ventoux.carnet"`.
- **GOTCHA**: Leave `productName: "Carnet"` and `version` unchanged.
- **VALIDATE**: `node -p "require('./apps/desktop/src-tauri/tauri.conf.json').identifier"` prints `com.ventoux.carnet`.

### Task 3: Make the prebuild verify scripts rename-proof (derive from app.json)
- **ACTION**: Edit both `apps/mobile/scripts/verify-audio-decoder-prebuild.sh` and `verify-notification-and-widget-prebuild.sh`.
- **IMPLEMENT**: Replace the hardcoded `PKG_PATH="us/beary/carnet"` with a derived pair near the top (after `MOBILE_DIR` is defined):
  ```sh
  PKG=$(node -p "require('$MOBILE_DIR/app.json').expo.android.package")
  PKG_PATH="${PKG//.//}"
  ```
  Then change the import-check strings to interpolate `$PKG`:
  - audio-decoder:84 → `check_main_app_contains "import ${PKG}.audiodecoder.AudioDecoderPackage" "import line present"`
  - notification:102 → `check_main_app_contains "import ${PKG}.notification.CaptureNotificationPackage" "import line present"`
- **MIRROR**: VERIFY_SCRIPT_HARDCODE pattern above; matches PR #23 review guidance.
- **IMPORTS**: relies on `node` (already used by build scripts) being on PATH.
- **GOTCHA**: `${PKG//.//}` is bash string replace (dots→slashes); keep `set -e` semantics — `node -p` must succeed, so run after the `MOBILE_DIR` resolution and before first use. Confirm no other line in the script re-hardcodes `us/beary/carnet`.
- **VALIDATE**: `bash -n <script>` parses; after Task 5, the scripts pass (see Task 6).

### Task 4: Fix the stale comment in withCaptureNotification.js
- **ACTION**: Edit `apps/mobile/plugins/withCaptureNotification.js` lines 28–31.
- **IMPLEMENT**: Update `us.beary.carnet.notification.*` in the comment to `<android.package>.notification.*` (or `com.ventoux.carnet.notification.*`) so it doesn't mislead. Code is unchanged.
- **GOTCHA**: Comment only — do not alter the surrounding `captureForegroundServiceKt`/derive logic.
- **VALIDATE**: `git grep -n 'us\.beary' apps/mobile/plugins` returns nothing.

### Task 5: Regenerate the Android native project
- **ACTION**: From `apps/mobile`, run a clean prebuild for Android.
- **IMPLEMENT**: `cd apps/mobile && npx expo prebuild --clean -p android` (regenerates `android/` from the updated `app.json`; plugins re-emit Kotlin under `com/ventoux/carnet/...`).
- **GOTCHA**: `--clean` deletes the existing `android/` first — safe here (gitignored, plugin-driven, no manual edits). The regenerated `debug.keystore` may differ from the prior one, but that's irrelevant: the new package is a brand-new app identity (no in-place update over a prior `com.ventoux.carnet`). iOS dir can be regenerated similarly on macOS later (`-p ios`); not required for the Android build.
- **VALIDATE**: `apps/mobile/android/app/build.gradle` shows `namespace 'com.ventoux.carnet'` and `applicationId 'com.ventoux.carnet'`; Kotlin exists under `android/app/src/main/java/com/ventoux/carnet/...` (e.g. `MainActivity.kt`, `notification/`, `audiodecoder/`, `widget/`).

### Task 6: Run the prebuild verify scripts
- **ACTION**: Execute both verify scripts against the freshly prebuilt project.
- **IMPLEMENT**: `cd apps/mobile && bash ./scripts/verify-audio-decoder-prebuild.sh && bash ./scripts/verify-notification-and-widget-prebuild.sh` (with `ANDROID_HOME=/home/user/Android/Sdk`).
- **GOTCHA**: These verify Kotlin was *emitted* and the MainApplication import/`add()` injection landed — they do NOT compile it (see `[[native-plugin-kotlin-verification]]`: only a full release build compiles the emitted Kotlin).
- **VALIDATE**: both scripts exit 0 and report the import lines present under the new package.

### Task 7: Release build + install as the new app
- **ACTION**: Build and install the renamed app.
- **IMPLEMENT**: `cd apps/mobile && ANDROID_HOME=/home/user/Android/Sdk ANDROID_SDK_ROOT=/home/user/Android/Sdk npm run android:release` (assembleRelease → `adb install -r`).
- **GOTCHA**: This installs `com.ventoux.carnet` as a NEW app *alongside* `us.beary.carnet` (different package → no replace, no shared data). A cold ~9 min build is possible if native caches were wiped by `--clean`; otherwise faster.
- **VALIDATE**: `BUILD SUCCESSFUL`; `adb shell pm list packages | grep ventoux` shows `com.ventoux.carnet`; launch via `adb shell monkey -p com.ventoux.carnet -c android.intent.category.LAUNCHER 1` opens the app.

### Task 8 (optional): Remove the orphaned old app
- **ACTION**: After confirming the new app works, uninstall the placeholder.
- **IMPLEMENT**: `adb uninstall us.beary.carnet` (destroys its sandboxed notes/audio — irreversible; user already accepted fresh start).
- **GOTCHA**: Only after verifying `com.ventoux.carnet` launches and is usable. This permanently deletes the old notes.
- **VALIDATE**: `adb shell pm list packages | grep beary` returns nothing.
